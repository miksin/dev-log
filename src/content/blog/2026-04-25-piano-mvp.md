## 概覽

今日完成 `miksin/piano` 的 MVP 開發，由 Hermes Agent 擔任 PM，協調 Engineer 與 QA subagents 進行開發。

## 完成功能

| 功能 | 技術 |
|------|------|
| 2-octave 鋼琴 UI | React + CSS |
| 電腦鍵盤對映 | `keydown` event listeners |
| Web Audio API 音頻引擎 | ADSR envelope、polyphony |
| MIDI 裝置支援 | Web MIDI API |
| 無障礙屬性 | `aria-label`、`role` |
| Unit Tests | Vitest（50 tests） |
| E2E Tests | Playwright（5 tests） |

## 技術決策

**不依賴外部音頻函式庫**：自製 `AudioEngine` 類別，使用 Web Audio API 的 `OscillatorNode` + `GainNode` 實作 ADSR 包絡。

**Playwright 與 Vitest 分離**：Playwright 的全域 `test()` 與 Vitest 衝突，透過更新 `vitest.config.ts` 的 `exclude` 解決。

**TypeScript strict mode**：全面啟用 strict，確保型別安全。

## 今日開發模式

放棄 cron job 定時觸發，改為使用者呼叫 → PM (Hermes) 自主推進。PR Review 在對話中即時完成，迭代速度更高。
